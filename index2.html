Популярные вопросы и ответы на них

Как задать редирект на собственную финишную страницу?

По умолчанию редирект происходит на finish.jsp, для того, чтобы задать редирект на собственную страницу надо указать successUrl таким образом

$('#container2').apishopsForm({ callback:1,
type:'inline', /*тип открытия [inline|modal]*/
form:'light', /*тип формы [normal|light|jquery-selector]*/
siteId:10221,
productId:632879,
price:1017.41, /*цена товара*/
priceRound:1017, /*округленная цена*/
successUrl:'http://sitename.ru/finish.jsp?id=', 
wpId:15743307
});

?id= в 'http://sitename.ru/finish.jsp?id=' является обязательным, т.к. наш плагин добавляет к ней номер заказа и финишная страница превращается в http://sitename.ru/finish.jsp?id=321321.

Как изменить кодировку

В случае необходимости использования скрипта другой кодировки вы можете подключить apishopsForm.utf8.js вместо apishopsForm.js

<script src="http://img.apishops.org/SinglePageWebsites/custom/js/apishopsForm.utf8.js"></script>

Куда можно спрятать код стандартной формы, что бы вывести её в Modal форме?

Вы можете разместить код своей формы в любой части страницы. Если тип формы в инициализации плагина задан как modal, плагин скопирует в себя код указанной формы и разместит его в modal окне. При этом, форма, на код которой Вы укажете сама по себе работать не будет, так как останется неинициализированной. В таком случае, чтобы задействовать эту форму на странице рекомендуем инциализировать плагин с типаом inline, тогда код вашей формы будет работать ещё и отдельно, не в modal окне

Где взять параметры siteId, wpId, productId

Для подключения и использования API вам необходимо знать siteId, wpId и productId

Внимание Обязательно, перед использованием формы для продажи того или иного товара добавьте его в свой проект
1 способ. Взять параметры из сгенерированного одностраничника

Эти параметры генерируются системой Apishops для каждого магазина-«одностраничника» и прописываются в HTML страницы магазина следующим образом:

Исходный код http://ham.apishops.ru/: 
<script type="text/javascript">
var siteId = 3016;/*ID сайта*/
var productId = 564022;/*ID товара*/
var price = 899.0;/*Цена товара*/
var priceRound = 899;/*Округленная цена товара*/
var wpId = 9587555; /*Внутренний артикул товара*/
var sourceParam = "";
var sourceRef = "";
</script>
2 способ. Взять параметры на сайта
siteId(id сайта) содержится во всех URL страниц управления проектом
productId(ID товара) можно взять из кладки «Ассортимент» главной страницы ApiShops (колонка ID)
wpId(внутренний артикул товара) можно найти в разделе «Товары»

Обзор

API позволяет подключать одну или несколько форм для оформления заказа или заказа обратного товара к любой страницы. Форм заказа может быть несколько для одного товара или сразу нескольких.

Для подключения и использования API вам необходимо знать siteId, wpId и productId (эти параметры генерируются системой Apishops для каждого магазина-«одностраничника» и прописываются в HTML страниц магазина) следующим образом:

Исходный код http://ham.apishops.ru/: 
<script type="text/javascript">
var siteId = 3016;/*ID сайта*/
var productId = 564022;/*ID товара*/
var price = 899.0;/*Цена товара*/
var priceRound = 899;/*Округленная цена товара*/
var wpId = 9587555; /*Внутренний артикул товара*/
var sourceParam = "";
var sourceRef = "";
</script>

Настоятельно рекомендуем подключать и использовать JS и CSS с серверов Apishops. В таком случае у вас всегда будет самая свежая версия API.

После установки (см. инсталляция) Вы можете подгрузить на свою страницу один из четерых типов форм:

1. Modal обычной формы

Подгружает все возможные параметры заказа в modal-окно, которое открывается при нажатии на кнопку

Пример:

Заказать товар! 
<script>
    $(document).ready( function() {
        $('#button1').apishopsForm({ callback:1,
                type:'modal', /*тип открытия [inline|modal]*/
                form:'normal', /*тип формы [normal|light|jquery-selector]*/
                siteId:10221,
                productId:632879,
                price:1017.41, /*цена товара*/
                priceRound:1017, /*округленная цена*/
                wpId:15743307
            });
    });
</script>
<a href="#" id=button1 class="apishopsFormButton apishopsFormBuy">
    <b>Заказать товар!</b>
</a>

Инсталляция Как установить

Этапы

Для подключения API необходимо подключить файл стилей, js файл библиотеки, и если на странице нету подключения jquery, подключить ещё jquery, а потом инициализировать одну из форм

1. Подключение стилей

Файл Подключение
apishopsForm.css <link rel="stylesheet" href="http://img.apishops.org/SinglePageWebsites/custom/css/apishopsForm.github.css">

2. Подключение JavaScript

Файл Подключение
JQuery
Внимание! Не стоит jquery подключать второй раз, если эта библиотека уже подключена ранее
<script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>
apishopsForm.js <script src="http://img.apishops.org/SinglePageWebsites/custom/js/apishopsForm.github.js"></script>

3. Инициализация плагина

Инициализация первой формы происходит через вызов jquery плагина $(..).apishopsForm({ callback:1,}). В качестве параметров используется один объект с параметрами.

Код:

$(document).ready( function() {
    $('#container2').apishopsForm({ callback:1,
            type:'inline', /*тип открытия [inline|modal]*/
            form:'light', /*тип формы [normal|light|jquery-selector]*/
            siteId:10221,
            productId:632879,
            price:1017.41, /*цена товара*/
            priceRound:1017, /*округленная цена*/
            wpId:15743307
        });
});

Параметры

Обязательные параметры

Параметр Значение по умолчанию Описание
siteId - id сайта
productId - id товара
wpId - внутренний артикул товара
type inline Значения inline(размещение внутри контейнера) или modal(размещение в модал-окне)
form normal Значения normal(обычная форма), light(упрощенная форма), html-код или jquery селектор. В случае html-кода или jquery селектора, которые указывает на html, который необходимо использовать, в качестве формы с полями заказа необходимо задать параметр inputs

Необязательные параметры

Внимание, продвинутый уровень!Часть необязательных параметров ниже необходимы только в случае использования собственных форм с собственныим именами для полей ввода или селектов
Параметр Значение по умолчанию Описание
successUrl /finish.jsp?id= url, по которому отпрвляется пользователь после успешной отправки заявки. В конец этого параметра добавляется id заказа.
modal <div class="apishopsModal apishopsAnimation apishopsSideFall"><div class="apishopsModalWindow"><div class="apishopsModalClose"></div><div class="apishopsModalContent"></div><div class="apishopsModalClose2"><a href="#">&#1079;&#1072;&#1082;&#1088;&#1099;&#1090;&#1100; &#1086;&#1082;&#1085;&#1086;</a></div></div><div class="apishopsModalOverlay"></div></div> jquery селектор, или html код модал окна. По умолчанию задается стандартный код для modal-окна, но используя для выбора jquery селектор можно или свой html можно задавать свой код для modal окна, в котором будут размещены поля
placement .apishopsModalContent контейнер внутри модального окна, где должен размещаться контент. Необходимо задавать в том случае, когда используется свой modal параметр
inputs Объект из параметров:
Параметр Значение по умолчанию Описание
address [name=apishopsFormAddress] jquery-селектор поля адреса
count [name=apishopsFormCount] jquery-селектор поля количества
delivery [name=apishopsFormDelivery] jquery-селектор поля типа доставки
email [name=apishopsFormEmail] jquery-селектор поля выбора типа доставки
fio [name=apishopsFormFio] jquery-селектор поля фио
payment [name=apishopsFormPayment] jquery-селектор поля выбора типа оплаты
phone [name=apishopsFormPhone] jquery-селектор поля номера телефона
region [name=apishopsFormRegion] jquery-селектор поля выбора региона
cost [name=apishopsFormCost] jquery-селектор контейнера, куда пишется итоговая цена
button [name=apishopsFormBuy] jquery-селектор контейнера или кнопки, нажатие по которой должно обрабатывать форму

Свои формы? Нет ничего проще!

Чтобы подключить наш плагин к своей форме достаточно задать своим инпутам наши "name" атрибуты apishopsFormCount, apishopsFormFio, apishopsFormPhone и apishopsFormAddress для ввода количества товаров, ФИО получателя, номера телефона и адреса получателя соотвественно

Внимание
«apishopsFormCount, apishopsFormFio, apishopsFormPhone и apishopsFormAddress» — это минимальный набор инпутов, который должен быть в вашей форме. Если в форме не будет хватать одного из этих инпутов — наш плагин не подключится к вашей форме и не сможет обрабатывать заказы с этой формы.
Что делать, если в вашей форме недостаточно инпутов?
Если в вашей форме недостаточно инпутв, например, не предусмотрено поле для адреса доставки, тогда вам надо добавить недостающее поле c type=hidden и name=apishopsFormAddress(если не предусмотрено поле для ввода адреса доставки). Примеры таких форм приведены ниже
